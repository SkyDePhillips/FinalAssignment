 List Tags

HTML provides three types of list elements for structuring content:

  • <ul> (Unordered List) − Displays items with bullet points.
  • <ol> (Ordered List) − Displays items with numbers or letters.
  • <dl> (Description List) − Used for terms and definitions.

Unordered List (<ul>)

An unordered list displays items with bullet points (default).

Syntax:

<ul>
<li>Apple</li>
<li>Banana</li>
<li>Cherry</li>
</ul>

Key Tags:

<ul> − Wraps the list items.

<li> − Represents each list item.

Ordered List (<ol>)

An ordered list displays items with a sequence (numbers, letters, or Roman numerals).

Syntax:

<ol>
<li>Step 1</li>
<li>Step 2</li>
<li>Step 3</li>
</ol>

Tags:

<ol> − Wraps the ordered list.

<li> − Represents each list item.

Common Attributes:

type − Specifies the numbering style:

<ol type="A"> <!-- A, a, I, i, 1 -->
<li>First</li>
<li>Second</li>
</ol>
start − Sets the starting number:

<ol start="5">
<li>Item 5</li>
<li>Item 6</li>
</ol>
 reversed − Displays list in reverse order:

<ol reversed>
<li>Step 3</li>
<li>Step 2</li>
<li>Step 1</li>
</ol>

Description List (<dl>)

A description list pairs terms with definitions, commonly used for glossaries or FAQs.

Syntax:

<dl>
<dt>HTML</dt>
<dd>HyperText Markup Language</dd>
<dt>CSS</dt>
<dd>Cascading Style Sheets</dd>
</dl>

Key Tags:

<dl> − Wraps the description list.

<dt> − Represents the term (definition title).

<dd> − Provides the description.

Example: Nested Lists

Lists can be nested inside each other.

<ul>
<li>Fruits
<ul>
<li>Apple</li>
<li>Banana</li>
</ul>
</li>
<li>Vegetables
<ol>
<li>Carrot</li>
<li>Broccoli</li>
</ol>
</li>
</ul>

Best Practices:

Use <ul> for unordered content (e.g., menus, lists).

Use <ol> when the order matters (e.g., steps in a process).

Use <dl> for definitions, key-value pairs, or FAQs.
